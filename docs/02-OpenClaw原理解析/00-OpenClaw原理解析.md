## 1. OpenClaw 是什么？

**OpenClaw** 是一个开源的 AI Agent 运行时平台，核心功能：

- 接收来自多种渠道的消息（QQ、Telegram、Discord 等）
- 调度 AI Agent 执行任务
- 管理定时任务（Cron）
- 维护对话记忆（Session）
- 支持多 Agent 协作

**类比理解：**
```
OpenClaw ≈ 机器人「大管家」+「调度中心」+「邮局」
- 大管家：管理 AI Agent 的行为
- 调度中心：决定什么时候做什么任务
- 邮局：从各个渠道收发消息
```

## 3. 核心模块详解

### 3.1 Channel（渠道层）

**作用：** 统一接收来自不同平台的消息

```
用户QQ消息  ──┐
              │
Telegram消息 ──┼──► Gateway ──► Agent Runtime
              │
Discord消息 ──┘
```

**支持的 Channel 类型：**
| Channel | 用途 | 配置参数 |
|---------|------|---------|
| `qqbot` | QQ 机器人 | appId, clientSecret |
| `telegram` | Telegram Bot | Bot Token |
| `discord` | Discord Bot | Bot Token |
| `wecom` | 企业微信 | CorpID, AgentID |
| `lightclawbot` | LightClaw | API Keys |
| `yuanbao` | 腾讯元宝 | AppKey, AppSecret |

**配置示例（qqbot）：**
```json
{
  "channels": {
    "qqbot": {
      "enabled": true,
      "appId": "1903135887",
      "clientSecret": "SI8yoeUKB2tkbSKC4wogYRKD6zsmgaUO"
    }
  }
}
```

---

### 3.2 Gateway（网关层）

**作用：** 消息的统一入口和分发中心

**职责：**
1. 接收来自各 Channel 的消息
2. 验证消息来源（防止伪造）
3. 将消息路由到对应的 Agent Session
4. 处理会话的生命周期（创建、维护、销毁）

**技术细节：**
- 监听端口：17080（可配置）
- 通信协议：HTTP WebSocket
- 认证方式：Token 验证

---

### 3.3 Agent Runtime（Agent 运行时）

**作用：** AI Agent 的执行引擎

**核心功能：**
```
用户消息
   │
   ▼
┌─────────────────────────┐
│  消息理解（Intent）        │
│  LLM 判断用户想要什么      │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│  Session 管理            │
│  创建/恢复/存储会话       │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│  Tool Executor          │
│  调用搜索/文件/API等工具 │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│  Response Generator     │
│  LLM 整合结果返回用户     │
└─────────────────────────┘
```

---

### 3.4 Cron（定时任务调度）

**作用：** 类似于 Linux 的 cron，但用于调度 AI Agent 任务

**原理：**
```javascript
// 定时任务的配置结构
{
  id: "每日音乐学习",
  cron: "0 8 * * *",        // cron 表达式：每天 8:00
  tz: "Asia/Shanghai",      // 时区
  message: "搜索100首流行歌曲...",  // 任务内容
  timeoutSeconds: 1800,     // 超时时间
  sessionTarget: "isolated", // 独立 session 执行
  delivery: {
    mode: "announce",      // 执行完后公告
    channel: "qqbot",       // 公告渠道
    to: "qqbot:c2c:用户ID"  // 发送给谁
  }
}
```

**cron 表达式详解：**
```
┌───────────── 分钟 (0-59)
│ ┌───────────── 小时 (0-23)
│ │ ┌───────────── 日 (1-31)
│ │ │ ┌───────────── 月 (1-12)
│ │ │ │ ┌───────────── 星期 (0-7，0和7都是周日)
│ │ │ │ │
* * * * *
│ │ │ │ │
│ │ │ │ └────── 周几（0=周日，1=周一...）
│ │ │ └──────── 月份
│ │ └────────── 日期
│ └──────────── 小时
└────────────── 分钟
```

**常用表达式：**
| 表达式 | 含义 |
|--------|------|
| `0 8 * * *` | 每天 8:00 |
| `0 9 * * *` | 每天 9:00 |
| `*/30 * * * *` | 每 30 分钟 |
| `0 10 * * 1-5` | 工作日 10:00 |
| `0 8,12,18 * * *` | 每天 8:00、12:00、18:00 |

---

### 3.5 Session（会话管理）

**作用：** 管理每次对话的上下文

```
Session 结构：
{
  id: "39b8e4f1-972e-4370-913a-b6fe684995af",
  key: "agent:main:qqbot:direct:3B3A517AFF...",  // 唯一标识
  channel: "qqbot",           // 来源渠道
  updatedAt: 1773969037128,     // 最后更新时间
  model: "MiniMax-M2.7-highspeed",  // 使用模型
  contextTokens: 200000,       // 上下文 token 数
  messages: [                   // 对话历史
    {role: "user", content: "你好"},
    {role: "assistant", content: "你好！有什么可以帮你？"}
  ]
}
```

**Session 生命周期：**
```
新建 Session ──► 接收消息 ──► 处理 ──► 保存 ──► 响应
                    ▲                        │
                    └────────────────────────┘
                         下一条消息（复用 Session）
```

**记忆压缩（Compaction）：**
当 Session 过长时，系统会自动压缩历史消息，保留关键信息。

---

### 3.6 Tool System（工具系统）

**作用：** 扩展 Agent 的能力，让它能做更多事情

```
OpenClaw 内置工具：
├── 搜索工具（web_search）
├── 网页获取（web_fetch）
├── 文件读写（read/write）
├── 代码执行（exec）
├── Git 操作（git）
├── 定时任务（cron）
├── 消息发送（message）
└── 云存储（cos upload）
```

**工具调用流程：**
```
Agent 想调用工具
    │
    ▼
 LLM 判断：「需要搜索网络」
    │
    ▼
 Tool Executor 执行 web_search
    │
    ▼
 返回搜索结果给 LLM
    │
    ▼
 LLM 整合结果生成回答
```

---

### 3.7 Memory（记忆系统）

**分层记忆架构：**
```
┌─────────────────────────────────────────┐
│         Long-term Memory（长期记忆）       │
│  MEMORY.md — 持久化关键信息               │
│  项目路径、用户偏好、角色设定             │
└─────────────────────────────────────────┘
                    ▲
                    │ 定期沉淀
                    │
┌─────────────────────────────────────────┐
│         Daily Memory（每日记忆）          │
│  memory/YYYY-MM-DD.md — 当日工作日志     │
└─────────────────────────────────────────┘
                    ▲
                    │ 自动记录
                    │
┌─────────────────────────────────────────┐
│         Session Context（会话上下文）      │
│  当前对话的上下文（200k tokens）         │
└─────────────────────────────────────────┘
```

## 5. 从 OpenClaw 学到的设计思想

### 5.1 消息总线模式
```
多渠道（QQ/TG/Discord）
    │
    ▼
统一消息格式
    │
    ▼
消息总线（Gateway）
    │
    ▼
统一处理（Agent Runtime）
```

### 5.2 插件化架构
```
Core（核心）
    │
    ▼ 加载插件
Channel 插件（QQ/TG/Discord...）
Tool 插件（搜索/文件/代码...）
Memory 插件（向量数据库...）
```

### 5.3 Isolated Session（隔离执行）
- 定时任务在独立的 Session 中执行，不干扰主对话
- 避免长时间任务阻塞用户对话

## 6. 我们要借鉴的设计

基于 OpenClaw 的架构，我们搭建平台时要借鉴：

| OpenClaw 的设计 | 我们平台的应用 |
|----------------|--------------|
| Channel 插件化 | 前端：统一的消息适配层 |
| Gateway 统一入口 | 后端：API 网关 |
| Cron 定时调度 | 后端：Celery 任务队列 |
| Session 管理 | 后端：Redis Session + PostgreSQL 存储 |
| Tool 系统 | 后端：工具注册 + 执行引擎 |
| 分层记忆 | 后端：向量数据库（Milvus）+ PostgreSQL |
| Isolated 执行 | 后端：独立 Worker 进程 |
